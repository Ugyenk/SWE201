# TaskFlow — Task Manager App

This is my SWE201 assignment: a single-domain CRUD app built with **React Native (Expo)**. I use it to keep track of tasks, sort them by status, and group them into categories.

---

## What the app does

TaskFlow is a personal task manager. I can sign in, create tasks, view and edit them, and delete them when I am done. Tasks belong to categories, and I can filter by status (`pending`, `in_progress`, `completed`) or search by keyword. Auth is token-based and the session stays logged in after I close the app thanks to AsyncStorage.

## Domain & entities

### Primary entity: Task

| Field | Type | Description |
|-------|------|-------------|
| id | string | Auto-generated by backend |
| title | string | Task title (required, 3–100 chars) |
| description | string | Optional notes (max 500 chars) |
| status | string | `pending` / `in_progress` / `completed` |
| categoryId | string | Foreign key → Category |
| dueDate | ISO string | Due date |
| createdAt | ISO string | Creation timestamp |

### Secondary entity: Category

| Field | Type | Description |
|-------|------|-------------|
| id | string | Auto-generated by backend |
| name | string | Category name (e.g. Work, Personal) |

## State management

I used **Zustand** for global state. It has less boilerplate than Redux and works well with TypeScript. For a single-domain app like this, it felt like the right balance.

- **What I store globally:** task list, category list, auth user + token, filter/search state
- **What I persist (AsyncStorage):** auth token, user object, last selected filter status
- **On launch:** `store.rehydrate()` loads saved values before the screens render

Custom hooks I wrote:

- `useForm` — form state and validation
- `useFetchTasks` — fetch tasks with cancellation
- `useFetchCategories` — fetch categories
- `useAuth` — sign in, sign up, sign out

## Backend

**Technology:** [mockapi.io](https://mockapi.io) — free mock REST API

### Endpoints

| Method | URL | Purpose |
|--------|-----|---------|
| GET | `/tasks` | List all tasks |
| GET | `/tasks/:id` | Get task detail |
| POST | `/tasks` | Create task |
| PUT | `/tasks/:id` | Update task |
| DELETE | `/tasks/:id` | Delete task |
| GET | `/categories` | List all categories |
| GET | `/categories/:id` | Get category detail |
| POST | `/categories` | Create category |
| PUT | `/categories/:id` | Update category |
| DELETE | `/categories/:id` | Delete category |

### Setting up mockapi.io

1. Create a free account at [mockapi.io](https://mockapi.io)
2. Create a new project
3. Add a **Task** resource with fields: `title`, `description`, `status`, `categoryId`, `dueDate`, `createdAt`
4. Add a **Category** resource with field: `name`
5. Copy the project base URL (e.g. `https://XXXXXX.mockapi.io/api/v1`)
6. Paste it into `.env` (see below)

## How to run it

### Prerequisites

- Node.js v18+
- Expo Go on your phone ([iOS](https://apps.apple.com/app/expo-go/id982107779) / [Android](https://play.google.com/store/apps/details?id=host.exp.exponent))

### Install

```bash
git clone <your-repo-url>
cd TaskManagerApp
npm install
```

### Environment

```bash
cp .env.example .env
```

Edit `.env` and set your mockapi.io URL:

```
EXPO_PUBLIC_API_BASE_URL=https://YOUR_PROJECT_ID.mockapi.io/api/v1
```

### Start the app

```bash
npm start
```

Then open it in **Expo Go** (scan the QR code), or press `a` / `i` / `w` for Android, iOS, or web.

## Known limitations

Things I would improve with more time:

- Auth is simulated locally in AsyncStorage — not a real backend auth endpoint
- Due date is a plain text field (`YYYY-MM-DD`) instead of a native date picker
- mockapi.io free tier can rate-limit (429) if you hit it too hard
- Categories can be created and deleted but not edited yet
- Some dates from the API may show as "Invalid Date" if the stored value is not a valid ISO string
